# Prompt 编写最佳实践

## 1. 指令要具体、明确

避免模糊的指令，尽可能量化要求。

| 不佳 | 优化 |
|------|------|
| "写一篇关于 AI 的文章" | "写一篇 800 字的中文科普文章，面向非技术背景的读者，介绍大语言模型的基本原理，使用类比帮助理解" |
| "分析一下这段代码" | "分析以下 Python 函数的时间复杂度和空间复杂度，指出可能的性能瓶颈，并提出至少两种优化方案" |

## 2. 使用分隔符区分不同部分

使用 XML 标签、Markdown 代码块或三引号分隔指令、上下文和输入数据：

```
请根据以下背景信息回答问题：

<context>
{背景信息}
</context>

<question>
{问题}
</question>
```

## 3. 明确指定输出格式

在 Prompt 中提前声明期望的输出格式，避免模型自由发挥：

```
请以以下格式输出：
1. 核心观点：（一句话总结）
2. 详细分析：（分点说明，每点不超过 50 字）
3. 建议措施：（按优先级排序）
```

## 4. 分步骤处理复杂任务

将复杂任务拆分为多个步骤，引导模型逐步完成：

```
请按以下步骤处理：
Step 1: 提取文本中的所有日期和地点
Step 2: 按时间顺序整理成时间线
Step 3: 为每个事件写一句简要描述
```

## 5. 提供示例（Few-Shot）

当任务模式难以用语言精确描述时，提供 1-3 个高质量示例：

```
输入："这个产品太棒了！"
输出：{"text": "这个产品太棒了！", "sentiment": "positive", "confidence": 0.95}

输入：{实际输入}
输出：
```

## 6. 设定边界和约束

明确告知模型什么不应该做：

```
约束条件：
- 不要使用专业术语，确保高中生能理解
- 回答长度控制在 300 字以内
- 不要提供医疗建议
```

## 7. 让模型先思考再回答

对于推理类任务，引导模型展示思考过程：

```
请先分析问题的关键要素，再给出最终答案。
```

## 8. 迭代优化

不要期望一次写出完美 Prompt，建议遵循以下迭代流程：

1. **初稿**：写出基本指令
2. **测试**：用 3-5 个不同输入测试
3. **分析**：记录失败案例和输出偏差
4. **修正**：针对问题调整指令、增加示例或约束
5. **回归**：确保修改没有破坏之前的成功案例

## 9. 版本管理

对重要 Prompt 进行版本管理：

- 记录每次修改的原因和效果变化
- 保留历史版本以便回滚
- 使用语义化版本号（如 v1.0.0 → v1.1.0）

## 10. 安全与合规

- 在 Prompt 中明确禁止生成有害内容
- 对敏感任务加入人工审核环节
- 避免在 Prompt 中硬编码 API Key、密码等凭证
- 注意 Prompt 注入风险，对用户输入进行过滤或转义
