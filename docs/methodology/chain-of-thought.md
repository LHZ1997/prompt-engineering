# Chain-of-Thought（思维链） prompting

## 核心思想

通过显式引导模型输出中间推理步骤，而非直接给出答案，从而显著提升模型在复杂逻辑、数学推理和多步骤任务上的表现。

## 适用场景

- 数学计算与证明
- 逻辑推理与判断
- 多步骤决策分析
- 因果推断

## 基本模式

在 Prompt 中加入以下指令之一：

```
Let's think step by step.
```

```
请逐步分析，并展示你的思考过程。
```

## 进阶技巧

### 1. Zero-shot CoT

无需示例，仅通过指令触发思维链：

```
问题：{question}

请逐步推理，先分析已知条件，再推导结论。
```

### 2. Few-shot CoT

提供包含推理过程的示例，引导模型模仿：

```
示例 1：
问题：一个农场有鸡和兔共 35 只，脚共 94 只。鸡兔各几只？
推理：设鸡 x 只，兔 y 只。x + y = 35，2x + 4y = 94...
答案：鸡 23 只，兔 12 只。

问题：{new_question}
请按照上述方式逐步推理。
```

### 3. Self-consistency CoT

多次采样并投票，选择出现频率最高的答案，提高稳定性。

## 注意事项

- 并非所有任务都需要 CoT，简单任务可能因冗余推理降低效率
- 推理步骤应清晰、有逻辑，避免循环论证
- 对于超长推理链，可结合 "总结中间结论" 指令防止上下文丢失
