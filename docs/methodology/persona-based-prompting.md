# Persona-Based Prompting（角色扮演提示）

## 核心思想

通过为模型设定一个具体的角色、身份或专业背景，激活模型在该领域的知识和语言风格，从而获得更专业、更贴合场景的输出。

## 适用场景

- 需要专业领域知识（法律、医学、金融）
- 需要特定语气或风格（资深编辑、耐心导师、严厉评审）
- 多智能体协作（让模型扮演不同角色进行辩论或协作）
- 用户培训与模拟（面试、谈判、咨询模拟）

## 基本模式

```
## Role
你是一位拥有 15 年经验的 {专业领域} 专家，擅长 {具体技能}。

## Background
你曾在 {知名机构} 工作，参与过 {代表性项目}。

## Personality
- 专业严谨，注重细节
- 善于用通俗语言解释复杂概念
- 回答简洁，直击要点

## Task
{具体任务描述}
```

## 角色设计要素

### 1. 身份定义

明确角色的职业、资历、专业领域：

- "你是一位资深 Python 性能优化工程师"
- "你是一位擅长写技术博客的开发者布道师"

### 2. 背景故事

补充背景信息增强角色可信度：

- "你曾在 Google 工作 10 年，负责搜索核心算法优化"

### 3. 性格与风格

定义沟通风格：

- 正式 vs 轻松
- 详细 vs 简洁
- 鼓励型 vs 批判型

### 4. 知识边界

明确角色的知识范围和限制：

- "你的知识截止于 2024 年 6 月"
- "你不提供医疗诊断建议，只提供一般性健康信息"

## 多角色协作模式

```
# 辩论模式

## 角色 A：支持者
你支持 {观点}，请从 {角度} 论证其优势。

## 角色 B：反对者
你反对 {观点}，请从 {角度} 指出其风险和不足。

## 规则
双方各陈述 3 轮，每轮不超过 200 字。最后由评委总结。
```

## 注意事项

- 角色设定应与任务高度相关，避免为了角色而角色
- 过于具体的虚构背景可能被模型"遗忘"，关键约束应在 Task 中重复
- 涉及敏感领域（医疗、法律）时，必须加入免责声明
